# 카탈로그 제목 2종 — 왜 이렇게 생겼나

카탈로그 각 행은 제목을 **두 가지**로 갖는다. 대상(누가 읽나)이 다르기 때문이다.

| | `title_geo` | `title_commerce` |
|---|---|---|
| 읽는 주체 | **AI 검색 엔진 / 봇** (ChatGPT·Perplexity·AI Overviews) | **사람 구매자** (쇼핑·검색·광고) |
| 구조 | 브랜드 + **canonical 모델명** + 유형 | 브랜드 + 상품명 + 성별 + 유형 + 색상 + 사이즈 |
| 단위 | **모델 1개** (색상·사이즈 무관하게 하나) | **SKU 1개** (색상 × 사이즈마다 별도 행) |
| 예 | `아디다스 F50 하이퍼패스트 클럽 축구화` | `아디다스 F50 하이퍼패스트 클럽 벨크로 아스트로 터프 키즈 축구화 핑크 200mm` |

---

## 1. `title_geo` — 왜 "브랜드 + canonical 모델명 + 유형"인가

**목적: AI 검색이 상품을 "하나의 엔티티(entity)"로 인식·인용하게 만든다.** (GEO = Generative Engine Optimization)

- **엔티티 앵커가 핵심.** 생성형 검색은 상품을 `브랜드 + 정규 모델명`으로 식별한다. 이 축이 흐리면 인용·매칭이 안 된다.
- **색상·사이즈·성별을 뺀다.** 이것들은 같은 제품의 *변형*이다. 제목에 넣으면 한 모델이 수십 개의 **거의 똑같은 제목**으로 늘어나 → 검색·LLM 입장에선 **thin/duplicate content**(내용 얇은 중복)로 보여 엔티티 신호가 희석된다. GEO 정석은 **정규 상품 1개 = 대표 페이지/제목 1개, 변형은 구조화 필드(schema.org `offers` 의 color·size)** 로 표현하는 것.
- **canonical 모델명으로 정제한다.** 원본 상품명엔 마케팅·디테일 수식어(`벨크로`·`아스트로터프`·`폴드오버 텅`)가 붙어 있다. 이걸 그대로 두면 임베딩이 흐려지고 사람이 실제로 검색하는 말과 멀어진다. 그래서 gpt-4o-mini로 **핵심 모델 라인만** 남긴다: `F50 하이퍼패스트 클럽 벨크로 아스트로 터프` → `F50 하이퍼패스트 클럽`.
- **유형(축구화/신발)은 남긴다.** "이게 무슨 제품인지"를 알려주는 카테고리 명사가 있어야 질의("아디다스 축구화")에 걸린다.

> 한 줄: **사람이 검색창에 칠 법한 정규 상품명** = 브랜드 + 깨끗한 모델명 + 무슨 제품인지. 변형은 제목이 아니라 데이터 필드로.

## 2. `title_commerce` — 왜 "브랜드 + 상품명 + 성별 + 유형 + 색상 + 사이즈"인가

**목적: 구매자가 "지금 살 수 있는 바로 그 물건"을 식별·결정하게 만든다.**

- **변형이 곧 정체성.** 구매자에게 "핑크 200mm"와 "블랙 260mm"는 **다른 상품**이다. 그래서 색상·사이즈가 제목에 들어가야 한다.
- **사이즈를 개별 전개한다.** style_code의 사이즈 리스트(200|210|…)를 **사이즈마다 별도 행(카탈로그)** 으로 쪼갠다. 이게 실제 **재고·구매·광고 노출 단위(SKU)** 와 일치한다. (그래서 264,219행 = 진짜 살 수 있는 조합 수.)
- **사람이 읽기 좋게 정규화한다.** 신발 숫자엔 단위(`200`→`200mm`), 색상은 한글(`Pink`→`핑크`), 가방 치수(`36*23*12cm`)는 제목에서 빼고 스펙으로, 아디다스 의류 `A/L`→`L`.
- **`공용`은 뺀다.** 저검색·노이즈 토큰이라 제목만 지저분해진다(성별 컬럼엔 남겨둠).
- **속성값은 개별 컬럼에도 그대로 노출.** 제목은 사람이 읽는 문장, 필터·정렬·매칭은 `gender/product_type/color/size/material/origin` 컬럼으로.

> 한 줄: **매대에 걸린 실제 상품 하나** = 브랜드 + 상품명 + 이 물건을 특정하는 속성(성별·유형·색상·사이즈).

## 3. 두 축을 나눈 이유 (요약)

| | GEO(발견) | Commerce(구매) |
|---|---|---|
| 단위 | 모델(정체) | SKU(변형) |
| 변형(색/사이즈) | **제외** → 구조화 필드로 | **포함** → 제목의 일부 |
| 이름 | canonical(피처 제거) | 원상품명 + 속성(사람 친화) |
| 최적화 대상 | AI/검색 엔티티 매칭 | 사람의 구매 판단 |

같은 원천 데이터를 **"어떻게 발견되나(GEO)"** 와 **"어떻게 팔리나(Commerce)"** 두 목적에 맞게 각각 최적화한 것이다. 하나의 제목으로 둘 다 하려면 반드시 한쪽이 나빠진다 — GEO엔 속성이 노이즈, 커머스엔 속성이 필수이기 때문.
